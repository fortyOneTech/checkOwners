# checkOwners

CODEOWNERS inference engine driven by git commit history with knowledge graph, expertise decay detection, ownership confidence scoring, team topology inference, and CI integration.

## Stack

- Python 3.11+
- CLI: Typer + Rich (colored output, progress bars, knowledge graph TUI)
- Git analysis: GitPython + subprocess (`git log`, `git blame`, `git shortlog`)
- GitHub integration: PyGithub (`GITHUB_OUTPUT` for Actions, PR review assignment API)
- Knowledge graph: networkx for contributor-file-team relationship modeling
- Config: PyYAML (`.github/checkowners.yml`)
- Packaging: hatch (PyPI); composite GitHub Action (`action.yml`)
- Testing: pytest + pytest-cov

## Commands

- `hatch run test`: run pytest suite with coverage
- `hatch run lint`: ruff check + mypy
- `hatch run fmt`: ruff format
- `checkowners analyze`: infer ownership from git history with confidence scores
- `checkowners generate`: write `.github/CODEOWNERS` with confidence-weighted owners
- `checkowners print`: print inferred owners to stdout with confidence scores
- `checkowners validate`: validate existing CODEOWNERS syntax only
- `checkowners drift`: compare inferred vs. current CODEOWNERS
- `checkowners notify`: POST webhook on drift
- `checkowners sync`: generate + commit CODEOWNERS
- `checkowners graph`: render knowledge graph of contributor-file relationships in terminal
- `checkowners graph --export dot`: export knowledge graph to DOT format for Graphviz
- `checkowners expertise <path>`: show expertise ranking for a specific path with confidence and recency
- `checkowners decay`: detect expertise decay (contributors who committed long ago but not recently)
- `checkowners topology`: infer team boundaries from commit clustering patterns
- `checkowners balance`: analyze PR review load distribution and suggest rebalancing
- `checkowners onboard <path>`: generate an onboarding path for learning a codebase area
- `checkowners bus-factor <path>`: calculate bus factor for each path (how many people must leave to lose knowledge)

## Architecture

```

  cli.py            # Typer app; all subcommands registered here
  analyze.py        # git log + git blame -> OwnershipMap with confidence scores
  generate.py       # OwnershipMap -> .github/CODEOWNERS writer (confidence-weighted)
  drift.py          # State machine: inferred vs. current diff -> DriftResult
  notify.py         # Webhook POST on drift events
  validate.py       # Syntax-only CODEOWNERS validator
  graph.py          # Knowledge graph builder: contributor-file-team relationships (networkx)
  expertise.py      # Expertise ranker: recency-weighted commit analysis per path
  decay.py          # Expertise decay detector: flag stale ownership based on time since last commit
  topology.py       # Team topology inference: cluster contributors by commit co-occurrence patterns
  balance.py        # PR review load analyzer: detect overloaded reviewers, suggest redistribution
  onboard.py        # Onboarding path generator: build learning paths from ownership chains
  busfactor.py      # Bus factor calculator: quantify knowledge concentration risk per path
  config.py         # PyYAML loader for .github/checkowners.yml
  models.py         # Dataclasses: OwnershipMap, DriftResult, ConfidenceScore, ExpertiseRank, TeamCluster, BusFactor, Config
  state.py          # ~/.checkowners/state.json reader/writer
tests/
  test_analyze.py
  test_generate.py
  test_drift.py
```

## Conventions

- Functional style throughout; no classes except dataclasses in `models.py`
- Type hints on every function signature; strict mypy (`--strict`)
- All CLI commands support `--json` flag for structured JSON output
- State file: `~/.checkowners/state.json` (auto-maintained, never hardcode path)
- Config file: `.github/checkowners.yml` (per-repo; loaded via `config.py`)
- Knowledge graph cache: `~/.checkowners/graph/<repo-hash>.json` (networkx serialized)
- Every write to `.github/CODEOWNERS` must include the machine-generated header:
  `# Generated by checkOwners. Do not edit manually.`
- Ownership is NOT binary; every path-owner pair has a confidence score (0.0-1.0) based on:
  - Commit recency (exponential decay with configurable half-life, default 90 days)
  - Commit frequency (total commits in lookback window)
  - Blame coverage (percentage of current lines attributed to contributor)
  - Review activity (PR reviews on files in the path, if GitHub API available)
- Default exclusions: `*.lock`, `dist/**`, `vendor/**`, `node_modules/**`
- Drift state machine modes: `commit`, `repo`, `both`
- Config defaults: `lookback_days: 365`, `min_commits: 3`, `top_n_owners: 3`, `confidence_threshold: 0.3`
- Expertise decay threshold: configurable, default 180 days since last commit to flag as decaying
- Team topology uses commit co-occurrence clustering (contributors who commit to same paths form implicit teams)
- Bus factor: paths with bus_factor=1 are flagged as critical risk (single point of knowledge failure)
- Onboarding paths follow the ownership graph from broad-ownership files to deep-expertise files
- Review load balance uses GitHub PR review API when available; falls back to git log author frequency

## Testing

- Every module has a corresponding test file in `tests/`
- Unit tests mock all subprocess calls (`git log`, `git blame`); never require a real git repo
- Integration tests in `tests/integration/` use fixture repos created with GitPython
- Knowledge graph tests use synthetic contributor-file networks
- Decay tests use time-manipulated commit histories
- Bus factor tests use known-topology fixture repos
- Coverage target: 85%+ on every module
- Run `hatch run test` before every commit; block merges below coverage threshold

## Do NOT

- No LLM or AI dependencies; checkOwners is pure-git inference only
- No write to `.github/CODEOWNERS` without the machine-generated header
- No external network calls except in `notify.py` (webhook POST), `balance.py` (GitHub PR API), and `action.yml` (GitHub API)
- No classes except dataclasses in `models.py`
- No skipped or missing type annotations
- No hardcoded file paths; use `pathlib.Path` and config-driven paths everywhere
- No modification of `state.json` schema without bumping the schema version field
- No networkx as a hard dependency; it must be an optional extra (`pip install checkowners[graph]`)
- No PyGithub as a hard dependency for core inference; GitHub API features are opt-in
- No confidence score below 0.0 or above 1.0; clamp all scores to valid range
- No binary ownership claims without accompanying confidence score
